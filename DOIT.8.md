Root cause in `bindFileExplorerChangesResizer` (`90_changes_editor.js:744`): the `pointermove`/`pointerup`/`pointercancel` listeners are bound to the 6px HANDLE and rely entirely on `setPointerCapture`, `done()` never calls `releasePointerCapture`, and the CSS handle (`.file-explorer-changes-resizer`, `60_editor_file_panels.css:624`) has no `touch-action`/`user-select` and only a 6px hit area. When you over-drag past the clamp the pointer leaves the 6px strip; capture + cleanup can leave the handle in a half-armed state and the tiny target is hard to hit again. FIX: (a) attach the drag `pointermove`/`pointerup`/`pointercancel` to `window` (canonical resizer pattern) so the drag survives leaving the strip; (b) call `handle.releasePointerCapture(event.pointerId)` in `done()`; (c) add `touch-action: none; user-select: none;` to the resizer and set a body no-select + `cursor: row-resize` while `.resizing-changes` is on; (d) enlarge the GRAB area to ~10–12px with a transparent `::before` overlay (negative block margins) while keeping the 6px visible line; (e) keep the existing hover/active highlight.
